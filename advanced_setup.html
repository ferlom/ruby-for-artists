# Cambios - Pulling

Si Vd. ha estado trabajando durante algún tiempo y nota que el repositorio del curso ha cambiado y desea ver el nuevo material:

  • Ver el "status" y asegurarse de que no tienes nada modificado

         git status

  • Tire de los cambios

         origin master git pull

# Forking (no la clonación)

Para guardar una copia de sus propias soluciones en su propio repositorio GitHub, tendrás que hacer un "Fork" del curso en Github. Esto te dará una nueva "url" de git de su propio "fork", que luego se podrá clonar desde el anterior. Por ejemplo, si su nombre es github lady ha-ha, su comando "clonador" será:

         git clone git :/ / github.com / ladyhaha / learn_ruby.git

Extraer los cambios de este nuevo "repo" es un poco complicado y, probablemente, debería pedir ayuda antes de hacerlo, pero he aquí las instrucciones disponibles:

  • Agregue una "referencia" al repositorio remoto original:

         git remote add Ultrasaurus git :/ / github.com / Ultrasaurus / learn_ruby

  • Baje el "repo" de la rama principal del repositorio remoto. Esto crearía una rama local llamada Ultrasaurus / master

         origin master git pull

  • Ahora puede fusionar la rama master del Ultrasaurus en su rama master

         Ultrasaurus git merge / master

  • Por último "push" otra vez a su propia rama master desde su rama origin del repositorio github remoto.

         origin master git push

Si Vd. quiere comprender todos estos "conjuros mágicos" de git, puede ver este gran screencast: http://blip.tv/file/4094854 (más en http://www.gitcasts.com)
