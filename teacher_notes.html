# Soluciones

¡ La rama soluciones ha muerto !; ¡ Viva el directorio solución !.

Para agregar una solución, debe poner a su archivo de Ruby una (s) y hacerlo en un directorio que este dentro del directorio del laboratorio que tenga el nombre de la solución `, por ejemplo,learn_ruby / hola / solución / hello.rb`.

Las soluciones sólo aparecen en el repositorio "master" (Formacion Primero-Test). No informa del estudiante generado (learn_ruby). Cuando ejecuta rake las pruebas se ejecutan a las soluciones, para asegurar que el código es correcto.

Asegúrese de no verificar en cualquier "cosa" en el directorio del laboratorio, ya que se verá por (y seguramente se copiara) los estudiantes.

Múltiples soluciones aún no están soportadas.

# Notas e instrucciones

Cada laboratorio tiene (o puede tener) un archivo index.md que los estudiantes pueden leer. Este es un buen lugar para el fondo, instrucciones y sugerencias para la resolución de los ejercicios.

Los archivos que terminan en . Md (Markdown) se convierten a HTML y se les da a. HTML sufijo `, y archivos que terminan en. Scss(Sass / SCSS) se convierten en CSS y se les da un. Css sufijo`. Los demás archivos se copian en el repositorio estudiante como es.

Los directorios de los laboratorio se vuelven numerados; por ejemplo hello será 00_hello.

Cada plan de estudios contiene un directorio activo, que es un buen lugar para poner los archivos compartidos como los js, los css y las imágenes. Puede hacer referencia a ellos utilizando .. en sus archivos en subdirectorios, por ejemplo,

     ! [logo] (.. / assets / logo.gif)

Funciona también en "modo vista previa" , aunque es un poco extraño, ya que .scss no se convierte en .css hasta que el curso se construye utilizando rake build.

También hay un directorio ubicuitus, cuyo contenido se copia en todas las formaciones y en cada directorio de laboratorio generado.

# Trucos

Estos son algunos buenos ajustes de configuración de git:

     git config --global alias.st status git config --global alias.ci commit git config --global alias.co checkout git config --global alias.br branch git config --global alias.unadd "reset HEAD" git config --global alias.l "log --oneline --decorate --graph"

 git config --global push.default matching

 git config --global color.branch=auto
 git config --global color.diff=auto
 git config --global color.interactive=auto
 git config --global color.status=auto

O bien, puede editar $ HOME / .gitconfig - usted debe tener secciones como esto:

     [color] branch = auto diff = auto interactive = auto status = auto [alias] st = status ci = commit co = checkout br = branch unadd = reset HEAD l = log --oneline --decorate --graph [push] default = matching
